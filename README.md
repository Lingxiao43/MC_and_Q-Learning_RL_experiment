# C++ 实现的强化学习：GridWorld 寻路

这是一个使用 C++ 实现的强化学习项目，旨在演示和比较 **蒙特卡洛 (Monte Carlo, MC)** 和 **Q-Learning** 两种经典的强化学习算法，以解决一个动态生成的 "GridWorld" (网格世界) 寻路问题。

本项目不仅包含了算法的基础实现，更侧重于展示 **高级的、自适应的 Epsilon (ε) 衰减策略**，使得智能体 (Agent) 的训练过程更加智能和高效。

## 核心概念展示
- **环境 (Environment)**: 一个可配置的 `GridWorld`，包含普通路径、陷阱和终点。
- **价值函数 (Value Function)**: 使用 Q-Table (一个三维数组) 来存储状态-动作对 `Q(s, a)` 的价值。
- **策略 (Policy)**: 经典的 ε-Greedy 策略，用于平衡 **探索 (Exploration)** 和 **利用 (Exploitation)**。
- **蒙特卡洛 (MC) 控制**: 通过完整的 Episode 回报来学习价值函数。
- **Q-Learning (TD 控制)**: 通过单步更新 (Bootstrapping) 来学习价值函数。
- **探索性出发 (Exploring Starts)**: 通过随机化每一轮的起点来保证充分的探索。
- **自适应学习率 (Adaptive Learning Rate)**: 使用“常数α”更新法，以适应策略的持续改进（非平稳性）。
- **智能 Epsilon 调度**: 两种先进的、基于性能反馈的 Epsilon 衰减机制。

## 文件结构与讲解

本仓库包含三个核心的 C++ 代码文件，每个都代表了一个不同层次或不同思路的强化学习实现。

### 1. `RL_MC_And_QLearning_My.cpp` (主要文件 & Q-Learning 核心)

这是本项目的主要文件，功能最全面，可以视为一个用于对比和实验的“瑞士军刀”。

**当前激活算法：Q-Learning**

该文件默认运行的是 **Q-Learning** 算法。其核心更新逻辑在训练循环中：
```cpp
// Q-Learning 的单步时序差分 (TD) 更新
G = reward + agent.gamma * agent.getBestActionQ(nextState); 
agent.Q[state.x][state.y][action] += 0.05 * (G - agent.Q[state.x][state.y][action]);
```
这完美遵循了 Q-Learning 的更新公式: $Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha [R_{t+1} + \gamma \max_a Q(S_{t+1}, a) - Q(S_t, A_t)]$

**✨ 独特亮点:**
*   **智能地图生成**: `pruneUnreachableAreas` 函数会在地图生成后，自动使用广度优先搜索 (BFS) **填埋所有无法到达终点的“死胡同”和“孤岛”**。这极大地提高了地图的质量，避免智能体在无效区域浪费大量训练时间。
*   **灵活的 Epsilon 衰减策略**: 代码中包含了多种（已注释的）Epsilon 衰减方案，如线性衰减、指数衰减、余弦衰减和分段衰减。这使其成为一个优秀的实验平台。
*   **完整的交互式演示**: 训练结束后，你可以在控制台中输入任意起点，观察训练好的智能体的最优路径。

### 2. `RL_MC_epsilon-greedy-2.cpp` (自适应MC：公式驱动衰减)

这是一个纯粹的蒙特卡洛 (MC) 实现，其精髓在于引入了一种**基于数学公式的自适应 Epsilon 衰减机制**。

**核心思想**: Epsilon 的值应该与智能体的**当前表现（成功率）**直接挂钩。
- **成功率高**: 说明智能体已经掌握了较优策略，应减少随机探索（降低 ε），专注于利用现有知识。
- **成功率低**: 说明智能体仍处于探索阶段，应保持较高的随机性（维持高 ε）来寻找更优路径。

这个思想通过以下公式实现：
$$ \epsilon = \epsilon_{initial} \times (1 - \text{SuccessRate})^k $$
其中 `k` 是一个可配置的幂指数，用于控制衰减的剧烈程度。

**✨ 独特亮点:**
*   **性能驱动的 Epsilon**: 彻底告别了“盲目”的、与训练轮次相关的衰减方式。Epsilon 的下降完全由智能体的实际学习进度决定。
*   **滑动窗口成功率 (Sliding Window SR)**: 为了让成功率的计算更平滑、更稳定，代码使用了一个高效的**环形数组**来实现滑动窗口，避免了单次成功或失败对 Epsilon 造成的剧烈抖动。
*   **动态窗口大小**: 窗口的大小会根据地图中“可出发点”的总数动态调整，确保了统计的合理性。
*   **完美早停 (Early Stopping)**: 当滑动窗口内的成功率达到 100% 时，程序会自动停止训练，节省了大量不必要的计算。

### 3. `RL_MC_epsilon-greedy-3.cpp` (自适应MC：平台期检测衰减)

这是另一个纯 MC 实现，它采用了一种更接近人类学习直觉的自适应策略：**平台期检测 (Plateau Detection)**。

**核心思想**: 在一个固定的 Epsilon 下持续训练，直到智能体的**学习陷入瓶颈（即成功率不再提升）**，然后才降低 Epsilon，帮助其“降维打击”，突破瓶颈。

**✨ 独特亮点:**
*   **瓶颈驱动的 Epsilon**: 模拟了“舒适区”学习法。智能体在当前探索水平下充分学习，直到潜力被榨干，再进入下一个更精细的“利用”阶段。
*   **耐心机制 (Patience)**: 通过 `patience` 参数，系统可以容忍成功率在一定窗口数量内的微小波动。只有当连续多个窗口的成功率都无法超越历史最高水平时，才触发 Epsilon 的衰减。
*   **稳健的决策**: 相比于公式法，这种基于平台期检测的机制对噪声不敏感，决策过程更加稳健，确保了每一次 Epsilon 的下降都是在深思熟虑后进行的。

## 如何编译和运行
你需要一个支持 C++17 (或更高版本) 的编译器，例如 `g++`。

1.  **编译**: 打开你的终端，进入代码文件所在目录，执行以下命令：
    ```bash
    # 编译主文件 (需要 C++20 以支持 <numbers>)
    g++ -o rl_main RL_MC_And_QLearning_My.cpp -std=c++20 -O3

    # 编译自适应MC文件 (C++17 即可)
    g++ -o rl_adaptive_formula RL_MC_epsilon-greedy-2.cpp -std=c++17 -O3
    g++ -o rl_adaptive_plateau RL_MC_epsilon-greedy-3.cpp -std=c++17 -O3
    ```
    *   `-O3` 是一个优化选项，可以显著加快运行速度。

2.  **运行**:
    ```bash
    ./rl_main
    ./rl_adaptive_formula
    ./rl_adaptive_plateau
    ```

3.  **交互**:
    *   程序启动后会要求你输入一些训练参数。
    *   训练结束后，会进入交互演示模式。
    *   你可以输入坐标 `(行 列)` 来设置起点，观察智能体的寻路过程。
    *   在演示动画播放时，可以随时按下 `q` 键来中断当前的演示。
    *   输入 `-1 -1` 退出程序。

## 核心知识点补充：常数α学习率的重要性
你可能会注意到，所有文件中的 Q-Table 更新都使用了一个固定的学习率 `alpha = 0.05`，而不是理论上收敛性更好的 `1/N`。

这是刻意为之，也是工程实践中的标准做法。因为强化学习的“控制”问题是一个**非平稳 (Non-stationary)** 问题——策略本身在不断进化。
- **`1/N` 平均法**: 给予了早期“愚蠢”策略和后期“聪明”策略相同的权重，导致智能体在训练后期变得**极其僵化，难以学习**。
- **常数α法**: 是一种**指数近因加权平均**，它会更看重**最近的经验**，并指数级地“遗忘”掉陈旧的经验。这使得智能体能持续适应新策略，保持学习的**可塑性**。
